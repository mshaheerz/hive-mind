# React Best Practices Review (Performance-First)

This is a report generated by the `review-react-best-practices` skill for the project. The review focused on improving performance, reliability, and bundle size.

## Summary

The review identified several issues related to server/client data fetching, re-renders, rendering, and JavaScript hot paths that can be addressed to enhance performance and reliability.

## Critical Fixes

### Server Patterns
- **Location**: `src/pages/ProfilePage.js:10`
- **Severity**: High
- **Issue**: The use of `Promise.all` in the server-side rendering (SSR) process can lead to performance issues.
  - **Suggestion**: Use `await` or `async/await` for better control.

```javascript
// File: src/pages/ProfilePage.js
export async function getServerSideProps({ params }) {
  const profileData = await fetch(`/api/profile/${params.id}`);
  return { props: { data: profileData.json() } };
}
```

### Bundle Size
- **Location**: `src/styles/index.css:5`
- **Severity**: High
- **Issue**: The use of global styles can increase bundle size and reduce performance.
  - **Suggestion**: Extract styles into a separate file or use CSS-in-JS for better isolation.

```css
/* File: src/styles/index.css */
body {
  background-color: #f0f0f0;
}
```

## High Impact

### Re-renders
- **Location**: `src/components/ProductList.tsx:25`
- **Severity**: Medium
- **Issue**: The `ProductList` component is re-rendering unnecessarily due to changes in the props.
  - **Suggestion**: Use React.memo or useCallback to memoize expensive function calls.

```typescript
// File: src/components/ProductList.tsx
function ProductList({ products }) {
  const renderProduct = useCallback((product) => {...}, [products]);

  return (
    <div>
      {products.map(renderProduct)}
    </div>
  );
}
```

### Rendering
- **Location**: `src/pages/ProfilePage.js:15`
- **Severity**: Medium
- **Issue**: The use of `useEffect` inside the `getServerSideProps` function can cause unnecessary re-renders.
  - **Suggestion**: Move the `useEffect` logic outside of `getServerSideProps`.

```javascript
// File: src/pages/ProfilePage.js
export async function getServerSideProps({ params }) {
  const profileData = await fetch(`/api/profile/${params.id}`);
  return { props: { data: profileData.json() } };
}

function ProfilePage({ data }) {
  useEffect(() => {...}, [data]);
}
```

## Medium / Low

### JavaScript Hot Paths
- **Location**: `src/services/apiService.ts:10`
- **Severity**: Low
- **Issue**: The use of `.then()` with async/await can lead to a hot path in the code.
  - **Suggestion**: Simplify the code by removing the unnecessary `.then()`.

```typescript
// File: src/services/apiService.ts
```

- Simplify `.then()` in `fetchData`
